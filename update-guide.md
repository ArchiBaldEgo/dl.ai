# Руководство: обновление проекта на сервере (Docker + Git)

Коротко, понятными командами: что делает и в каком порядке выполнять. Не углубляюсь в подробности флагов — только практичные шаги.

---

## Общий порядок обновления
1. Перейти в каталог проекта
   - `cd /path/to/project` — перейти в папку с репозиторием
2. Проверить текущую ветку и состояние
   - `git status` — показать незакоммиченные изменения
   - `git branch` — показать текущую ветку
3. Переключиться на нужную ветку
   - `git checkout main` — переключиться на ветку main (или другая ветка деплоя)
4. Получить последние изменения из удалённого репозитория
   - `git fetch --all` — получить обновления из удалённого репозитория
   - `git pull origin main/или же другая` — внедрить изменения из ветки main
5. Построить и перезапустить контейнеры (см. раздел «Сборка и запуск Docker»)

---

## Работа с Git — полезный порядок
- `git status` — текущее состояние рабочей копии
- `git pull origin BRANCH` — подтянуть и смержить изменения из удалённой ветки BRANCH
- `git fetch origin` — получить ссылки на удалённые обновления без слияния
- `git checkout BRANCH` — переключиться на ветку BRANCH
- `git merge origin/BRANCH` — влить изменения из удалённой ветки явно
- `git log --oneline -n 10` — посмотреть последние 10 коммитов
- `git diff` — посмотреть отличия
- `git reset --hard HEAD` — откатить локальные незакоммиченные изменения (ОСТОРОЖНО!)

## Сборка и запуск Docker (общие варианты)
### Если используется docker-compose (prod)
- `docker compose -f docker-compose.prod.yml pull` — опционально подтянуть свежие образы
- `docker compose -f docker-compose.prod.yml build --pull` — пересобрать образы с учётом Dockerfile.prod
- `docker compose -f docker-compose.prod.yml up -d --remove-orphans` — поднять контейнеры в фоне
- `docker compose -f docker-compose.prod.yml down` — остановить и удалить контейнеры/сеть/тома (по необходимости)

---

## Примерный сценарий обновления (docker-compose.prod.yml)
1. `cd /path/to/project`
2. `git checkout main/или же другая`
3. `git pull origin main/или же другая`
4. `docker compose -f docker-compose.prod.yml up -d`
5. `docker ps` — убедиться, что контейнеры запущены
6. `docker compose -f docker-compose.prod.yml logs -f` — мониторить логи (при необходимости)

---

## Полезные команды (коротко)
- `docker ps` — увидеть запущенные контейнеры
- `docker logs -f CONTAINER` — смотреть логи контейнера
- `docker compose -f file.yml ps` — статус сервисов
- `docker compose -f file.yml exec SERVICE sh` — попасть внутрь контейнера (или bash)
- `docker image ls` — список образов
- `docker system prune -f` — очистить неиспользуемые ресурсы (осторожно)
- `git stash` — временно убрать локальные изменения
- `git stash pop` — вернуть сохранённые изменения
- `ssh user@host` — подключиться к серверу
